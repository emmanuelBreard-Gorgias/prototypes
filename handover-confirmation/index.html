Design notes

Why radio cards, not a toggle

Both behaviors are first-class. A toggle hides what "off" means and forces merchants to learn it from a tooltip.

Default for new accounts

Ask the shopper to confirm. This matches the new behavior shipping in M1/M2 and reduces the handover rate (~5% relative drop in early data).

Default for existing accounts

Already on the feature: keep Ask. Not yet on the feature: default to Direct (current behavior). Don't auto-opt-in, that caused the complaints.

Always-direct exceptions

  • Shopper explicitly asked for a human
  • AI Agent can't reply (pipeline error / Reply Generation QA failed)

Surfaced in the footnote so merchants don't file bugs about "I want a human" skipping the prompt.

Storage

Stored at meta.preferences.handover_confirmation as 'ask' | 'direct'.
